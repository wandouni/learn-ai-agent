Agent 核心
run_agent() — 构建消息 → 发给 DeepSeek
每次调用 run_agent，都会先把三条消息打包：系统角色设定 + 用户问题，然后连同工具菜单一起发给 DeepSeek，等待它"思考"。

def run_agent(user_query):
    messages = [
        {"role": "system",  "content": SYSTEM_PROMPT},
        {"role": "user",    "content": user_query},
    ]

    while True:  # ← Agentic Loop 开始
        response = client.chat.completions.create(
            model="deepseek-chat",
            messages=messages,
            tools=TOOLS,
            tool_choice="auto",  # 让 AI 自己决定
        )
发出去的消息包
system 天气播报员角色设定
user 北京今天天气怎么样？
tools get_weather / get_aqi 说明书
→HTTP
DeepSeek 返回
finish_reason: "tool_calls"
调用 get_weather("北京")
调用 get_aqi("北京")
05
Agentic Loop 核心
执行工具 → 把结果喂回去 → 循环
🤖
DeepSeek 决定调用工具
finish_reason == "tool_calls"，取出 msg.tool_calls 列表
⚙️
execute_tool() 分发执行
查字典 TOOL_HANDLERS → 调对应函数 → 真实发起 HTTP 请求 → 返回 JSON 字符串
🌐
外部 API 返回真实数据
wttr.in → 32°C、晴天、湿度45% | WAQI → AQI=127、轻度污染
📨
结果追加进 messages
role: "tool" 消息追加，带上 tool_call_id 配对，再次调用 API
✅
DeepSeek 综合数据，返回最终答案
finish_reason == "stop" → 跳出循环，返回 msg.content
🍳
厨师点菜类比：你（用户）点了"跑步建议"，大厨（DeepSeek）说"我需要天气和空气质量"，服务员（execute_tool）去厨房取来食材，大厨把菜做好端上来。整个过程 while True 就是"大厨工作中"的状态，直到菜做完才停。
06
输出结果
打印 + 返回最终回答
finish_reason == "stop" 时，循环结束，把 AI 的回答打印出来并返回。 如果遇到 length（太长被截断）或 content_filter（内容过滤），则返回错误提示。

elif finish_reason == "stop":

⑧ 第2次请求 携带数据再次问 DeepSeek completions.create() ✅ DeepSeek API
⑨ 完成 AI 综合数据，输出最终建议 finish_reason="stop" —
💡
核心概念：Agentic Loop
while True 循环不是"死循环"，而是让 AI 可以反复使用工具直到任务完成。 每一轮：AI思考 → 要工具 → 我们执行 → 把结果告诉AI → AI继续思考……直到 AI 说"我说完了"（stop）才退出。 这是所有 AI Agent 最核心的工作模式。

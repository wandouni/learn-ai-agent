// 第四步 · 代码调用链
谁在什么时候调用了向量化
向量化发生了两次：写入时（存文档）和读取时（查询），都是 ChromaDB 自动触发的
1
server.py — 创建 collection，绑定 embedding 函数
collection 是 ChromaDB 里存数据的容器，创建时把向量化工具"绑"进去，之后读写都自动用它
# indexer.py · get_or_create_collection() from chromadb.utils.embedding_functions import SentenceTransformerEmbeddingFunction ef = SentenceTransformerEmbeddingFunction( model_name="paraphrase-multilingual-MiniLM-L12-v2" # ← 支持中文的多语言模型 ) collection = client.get_or_create_collection( name="knowledge_base", embedding_function=ef, # ← 向量化工具绑在这里 metadata={"hnsw:space": "cosine"}, # ← 用余弦距离 )
2
写入时：indexer.py 调用 upsert，自动向量化
你只传原文，ChromaDB 内部调用 ef(chunks) 得到向量，把原文 + 向量 + metadata 一起存进去
# indexer.py · index_document() chunks = _chunk(text) # 切成 500 字的段落列表 collection.upsert( ids=[f"{filename}__{i}" for i in range(len(chunks))], documents=chunks, # ← 只传文字，向量化在内部自动发生 metadatas=[{"source": filename, "chunk_index": i} for i in range(len(chunks))], ) # upsert 内部干了什么： # 1. ef(chunks) → [[0.12, -0.34, ...], [0.87, 0.05, ...], ...] # 2. 把向量 + 原文 + metadata 存进 ChromaDB 的 HNSW 索引
3
查询时：retriever.py 调用 query，自动向量化 + 检索
查询文本也先向量化，然后在 HNSW 索引里找"坐标最近"的段落，返回原文
# retriever.py · search() results = collection.query( query_texts=[query], # ← 传查询文字，自动向量化 n_results=top_k, include=["documents", "metadatas", "distances"], ) # query 内部干了什么： # 1. ef([query]) → [[0.09, -0.31, 0.84, ...]] 向量化查询 # 2. HNSW 找出余弦距离最小的 top_k 个向量 # 3. 取出对应的原文段落 + 距离分数返回 similarity = 1.0 - dist # cosine distance → similarity（越大越相似）
4
SentenceTransformer 模型本身做了什么
真正的向量化在这里发生，paraphrase-multilingual-MiniLM-L12-v2 是一个 1.17 亿参数的神经网络，输入文字，输出 384 维向量
# 这是 SentenceTransformerEmbeddingFunction 内部干的事（你不需要写） from sentence_transformers import SentenceTransformer model = SentenceTransformer("paraphrase-multilingual-MiniLM-L12-v2") # 模型首次使用时从 HuggingFace 下载（~270MB），之后缓存本地 vectors = model.encode(["连续登录失败", "login failed"]) # → array([[0.12, -0.34, 0.87, ...], shape: (2, 384) # [0.11, -0.33, 0.86, ...]]) ← 两个词向量几乎一样！
// 工具清单
用了哪些库，各自干什么
📦
sentence-transformers
加载神经网络模型，把文本编码成向量。

我们用的模型：paraphrase-multilingual-MiniLM-L12-v2
12层 Transformer，输出 384 维，支持 50+ 语言，包括中文。
🗄️
chromadb
本地向量数据库，负责存储和检索。

提供 SentenceTransformerEmbeddingFunction 包装器，把 sentence-transformers 和 collection 绑在一起，让 upsert/query 时向量化自动发生。
🔍
HNSW 索引算法
ChromaDB 底层用的检索算法（Hierarchical Navigable Small World）。

在百万级向量里找最近邻，复杂度 O(log n)。我们配置了 cosine 距离度量。
🤗
HuggingFace Hub
模型下载来源。首次使用时 sentence-transformers 自动从这里拉取模型权重文件（约 270MB），之后缓存在本地 ~/.cache/，不需要联网。
paraphrase-multilingual-MiniLM-L12-v2 · 384-dim · cosine similarity · ChromaDB HNSW
